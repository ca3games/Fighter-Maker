    2D Fightermaker 2nd - Getting Started



    Alright, after extracting the program to your desired directory you can get started.  Open up the editor program, the icon for which looks like this.



    If this is the first time you've used the program you will come to a blank screen as well as four tabs at the top,

    File, Test Play, Options, and Help.

    To get started, click File, and select New Game.  A prompt window should come up asking you to input a name for your project.  Mine will be titled "Tutorial Project".



    After you enter a name for the project, click OK.

    At this point, you will be shown the Game System tab.  This tab has numerous settings for the "engine" your fighting game will run on.  The Basic tab is discussed in this section.  The other tabs are discussed here.



    The Offset checkbox allows you to toggle colliding attacks (via Samurai Shodown) on or off.

    Stiff Time actually refers to the game pause that happens under specific conditions.  Hit means the pause when an attack connects, Guard means the pause when an attack is blocked, and Offset is a pause when two attacks collide. (This is usually a very heavy pause, i.e. higher value, to enhance dramatic effect.)

    The various checkboxes refer to:

    Use Story Mode - With this box checked, your game has a one player "Story Mode".  Setting up a story mode is explained here.

    Use Vs Mode - With this box checked, your game supports two player Vs. mode.  (I cannot stress how necessary this is... one player fighting games are teh suck.)

    Use Team Mode - Checking this box allows your game to use a Team Vs. mode, somewhere along the lines of the King of Fighters, or Virtua Fighter series. (The latter being closer, due to the fact that there is no "Team Story Mode".)

    The Editor Won't Read the Game File - Checking this box is a must to protect your work.  This option does not allow the *.kgt file to be read by the editor after it is compiled.

    Numbers Are Shown On the Life - Checking this box places numbers next to each member of a team during Team mode. 

    Cursor Stays Until Game Starts - From all experimentation I've found that this option sets whether or not your mouse pointer is active at startup of your game.  Not entirely positive however.

    The Demo Sel area is fairly self explanatory, but for sake of completeness, here are it's functions.

    Title - This is where you place the demo for your Title screen.

    Character Selective 1P - This is the demo for the character select screen in Single Player mode.

    Character Selective VS Single - This is the character select demo for Multiplayer Single Versus mode.

    Character Selective VS Team - This is the character select demo for Multiplayer Team Versus mode.

    Game Over - This is the Continue screen demo shown after a Game Over in Single Player mode.

    Opening Demo - This is the opening cinematic demo shown at game start.

    Main Menu - Test Play




    The test play tab allows you to test your game's playability via a simple interface.  The Test Play menu appears as such.



    The drop bars at the top are your character selection spaces.  Click the dropdown button to select from a list of characters in the game. (Player 1 is default palette, Player 2 is secondary palette by default.)

    The second drop bars are player behaviors.  The selectable behaviors available are:


    Player - This is the default setting.  This means that the character selected is played by a human via keyboard or joystick.

    CPU - This setting means that the character selected is operated via a set of artificial intelligence instructions.  The process for setting up AI is discussed here.

    Imitate - This setting makes the character that is not operated by Player or CPU imitate the actions taken by the acting fighter.  (I.e. - Player throws a jump kick, so does the CPU, etc.)

    Guarding - This setting places the character in a state of auto-guard.  All attacks are blocked (save throws) and the character cannot act otherwise.

    Jumping -  This setting is similar to Guarding, except the character auto-jumps.  This setting is generally good for testing juggles and attacks of that manner.


    The BG drop bar allows you to select a stage to fight in.

    Start Play refers to what section of the game is loaded to run.  Either From Title, or From Battle.

    From Title means the game will be loaded to the Title screen, allowing for you to test out each mode as if the game were being run as a stand alone executable.

    From Battle loads up a single bout match between the two selected characters on the stage that was selected in the BG tab.


    The  Game Speed slider allows you to set the speed the game runs at.  The default setting is 10, running the game at normal speed.

    Game Time allows you to set the number of seconds a fight runs, from 0 - 99.  Setting the slider to 0 allows for an infinite amount of play time.

    The Setup section allows you to enable various debug settings to view during the fights.  While they have no true impact on gameplay, they give you information on how the game runs.  You can often find solutions to a few glitches with these settings.

    Status - With this box checked, various status objects and meters will be shown on screen, this includes collision detection, current number of wins and losses, remaining hit points (in numeric value), character status (either Active, or Inactive, as well as standing, crouching or in air), as well as how many objects are currently on screen.

    Show FPS - With this box checked, a number will be shown on screen measuring frame output performance of your game.

    Joystick - With this box checked, your game will be able to use a joystick or control pad during play.  In my opinion, this should never be left off.

    Other Status - Unfortunately, as of this writing, I have not seen what this feature actually does.  It does not appear to display anything on screen.

    That about does it for Test Play.

    Animation - Adjusting Frames


    You'll want to align your frames well so that your animations hold together and frames don't needlessly slide back and forth.

    This is accomplished using the Axis settings in the image properties area.



    The first value is the frame's X (Horizontal) value, the second is the frame's Y (Vertical) value.
    Simply adjust the values until you're happy with them.  Continue the operation on each frame of your skill.

    This is also a good time to mention that the ANI button () is used to preview your animations.  A new window will pop up allowing you to see how everything looks.

    After this is finished, you'll need to adjust the timing of each frame.  You can adjust this in the Image properties area as well.



    The Wait value is the timing of your frames.  A value of 100 is roughly equal to one second, whereas a value of 0 is a constant loop.  The default value is a setting of 5, which is almost always too fast.  Adjust the value up or down to your liking.  (You may need to wait a bit to adjust the speed for your jumping animations)

    The I section is a number value representing the image used in the current frame, every time you add an image, an extra number is available for use.  Think of it like a storage area.  This allows you to reuse frames in other animations without uploading the same one over and over again.

    Complete this process with all your basic animations.

    Creating Your Fighter - Dealing With Movement



    Alright.  You should have all your animations compiled by this point. (If you haven't done so, check this section.)

    First, of course, are the basics.  I will be explaining how exactly all your movement options work, then move on to the application of these basics.

    Go to your Forward skill, then right click on the first square.  A long dropdown options menu should pop up. 

    Click on Add > Move.  A small box with an M in it should appear (), similar to the Image box.  This is your movement box.  This will allow your character to move via specified values.  At the left side, where your Image properties usually are, is the Movement Properties section.



    This is the area of the editor that will allow you to edit the values that create movement.  The values are:

    Move X and Y - These values represent actual movement.  X is of course, horizontal movement, and Y is vertical movement.  The higher the values, the more extreme the movement.  If these numbers are a positive value, X will move forward, and Y will move down.

    Gravity X and Y - Similar to the Move X,Y values, these control the degree of horizontal and vertical movement.  The difference is, these apply to gravity rather than actual movement.  They create a drag on the character in a direction specified by their numerical values.  A negative in the X value will result in a backward pull on your characters movement (this is good for slowing a character down during a run, for example.)  A negative in the Y value will result in an upward pull, basically sending your character toward the sky.  There aren't many uses for a negative Y gravity value.  The opposite is true as well.  Positives in either value will result in either a forward pull on your character, or a downward pull on your character (One of the most useful applications of the gravity field).

    Now, onto the applications of movement.  Basic movement (walking and jumping) are the only movements covered in this section.  For other applications, (dashing, etc.) check the Advanced Techniques section.

    Dealing With Movement - Creating Motion



    Now that you understand the basics, it's time to get down to creating.  Walking will be first.  Go to your Forward skill if you're not there already, and click on the box to bring up your movement properties.

    Now, there's a bit of variation necessary here.  The actual speed of movement you want really depends on the size of your character, and what seems to work best for them.  I typically stick to a value between +2.50 and +5.00 for any and all ground movement.  However, it seems in the case of Rockman, that this will not work in all cases.  His walking works best at a value of +4.25, which almost tops off my personal maximum of +5.00.  As I said it really depends on your character.

    Anyway, after you've clicked on your box, left click on the Move X value.  This will increase the numbers.  Set it to something comfortable and then move to your Backward skill.  Place an Add > Move box at the beginning of the skill if you haven't already.

    Characters in fighting games don't typically retreat as fast as they advance, so you'll need to set this to a slightly lower value (NEGATIVE value, that is) proportional to the original forward movement.  For Rockman, I set his value to -3.90, but it's all up to personal preference.

    Click on the button to preview your movement.  You should notice the background grid is moving.  This allows you to check if your values have been set correctly and that your character isn't doing some bizarre moonwalk.



    After you get everything comfortable, we can move on to Jumping.

    First, go to your Jump Up skill.  

    Here is where we need a little more variation.  I typically have at least one frame at the beginning that shows the character preparing his jump.  If you have one of these, right click the second image in the skill and Add > Move.  This will make sure the character is still grounded while preparing his jump.  If you do not have a frame like this, add your box at the beginning of the animation.  It won't cause any problems if you use either method, it's merely a matter of preference.

    The process for creating movement is very similar, except the majority of the movement is handled by the Move Y value, and this time we will also be using the Gravity Y value.

    Here you can use a pretty strict standard as far as movement settings.  I typically set my vertical values anywhere between -10.00 and -11.50.  This works in this case.  I set Rockman's Move Y value to -11.50, exactly at my limit.  I cannot stress enough that the Move Y value for a jump has to be a NEGATIVE value... It won't work correctly otherwise.  I typically stay away from anything over -11.50 because the jumps start to get way too floaty at that point.

    Anyway, now we need to apply gravity to the jump to bring your character back to the floor.  I ALWAYS set my Gravity Y value to +0.60 when dealing with jumps.  ALWAYS!  It's a good comfortable falling speed for this application.

    After you're finished with that, copy and paste the box from Jump Up into both your Front Jump and Back Jump skills, this gives us a base to edit for each of those skills.

    Leaving both Y values identical, in your Front Jump place a positive value in your Move X value.  Keep it fairly similar to your Forward walking movement, or slightly less (I set Rockman's to +4.40).



    Do the same thing with your Back Jump skill, except replace the Move X value with a negative value.  This value can be identical to or slightly more than your Move X value from your Walking Backward skill. (Rockman's is -4.00)



    That about does it for your movement.  However, as an aesthetic choice you can create a Landing skill to occur after your jumps hit ground. 

    Add a new skill to your list called Landing or something to that effect, then create a landing animation.  Make sure it causes a noticeable pause after the jump lands. (Typically an animation with a Wait time total of 6 is a good amount of time.)

    Then, go back to your Jump Up skill and right click the first box in the skill.  Hover over Add > Condition Fork, and a sub-menu should pop up.  Click on [DS] Add > Detect Skill.  A box with a DS should appear in your skill ().  This allows you to set a condition telling the character to go to a specific skill when certain conditions are met.



    The conditions that are able to be used with this object are

    Not - This is a placeholder - It does nothing.

    Landing - This condition is met when a character hits the ground after being in the air for any reason.

    Attack Hits - This condition is met when an attack you have thrown lands successfully.

    Defending - This condition is met when you block an attack.

    Hit the Wall - This condition is met when your character is touching either outside "Wall" of a stage while moving.

    In Offset - This condition is met when both your and your enemy's attacks collide.

    While do Throw - As of this writing, I am unsure of what this does.  I've never particularly found a use for it.

    After you select one of these options, the dropdown below will light up.  The dropdown bard allows you to select the skill you wish to move to, while the number beside it allows you to select a specific frame in the skill to go to.

    Set this to Landing 0.

    Now, every time you land after jumping up, it should go to this animation.

    Copy and Paste the box to both your Front Jump and Back Jump skills, and you're done!

    Creating Your Fighter - Hit Junctions and Common Images



    Okay, we're into important stuff.  This is the FIRST thing you should do when starting a project, as it will make your life so much easier during character creation. 
    The reason for this is that after you've set up your hit junctions/common images list, when you add a new character to your game, they will be pre-loaded with each of the hit junction skills necessary to work in that project. These skills are of blank, of course, but it means you don't need to add a thousand different skills to a character by hand.  It will also set up your hit junctions for you in your character's Hit Junction tab.  That way all you have to do is select the junction you want to use when setting up attacks.

    Hit junctions and Common Images allow you to create the reactions characters have to being hit by attacks, be they punches, kicks, throws, or off the wall special attacks, here's where the basic fundamentals lie.

    You may have already noticed you've got some hit junction animations available to you in the Skill Editor,

    Standing Wk Hit
    Crouching Wk Hit
    Jumping Wk Hit
    Standing Wk Guard
    Crouching Wk Guard
    Jumping Wk Guard
    Standing Sg Hit
    Crouching Sg Hit
    Jumping Sg Hit
    Standing Sg Guard
    Crouching Sg Guard
    and Jumping Sg Guard

    These are your general basic hit junctions, they are animations set up so characters can get hit by weak and strong attacks while either standing, jumping or crouching.  They also have guarding animations to allow the character to block.

    This is all well and good, but we know any good fighting game has to have MANY more of these hit junctions to function correctly.  I have a basic list of these that I use for everything, I call it the "FM06 Standard Junctions" list, referring to the Fighter Maniax website that my friend and I run.

    This list is as follows (It's pretty long... so I've grouped some of them together.  But any time you see a word group separated by a comma, those are each separate hit junctions.)

    Hit Weak Standing, Crouching, and Air
    Hit Weak Crouched Attack
    Slashed Weak Standing, Crouching, and Air
    Slashed Weak Crouched Attack
    Hit Middle Standing, Crouching, and Air
    Hit Middle Crouched Attack
    Slashed Middle Standing, Crouching, and Air
    Slashed Middle Crouched Attack
    Hit Strong Standing, Crouching, and Air
    Hit Strong Crouched Attack
    Slashed Strong Standing, Crouching, and Air
    Slashed Strong Crouched Attack
    Face Plant
    Lit on Fire Ground, Air, and Launched
    Hit Special Attack Standing, Crouching, and Air
    Hit Super Attack Standing, Crouching, and Air
    Knocked Down, Light, Middle, Heavy and Special Attack
    Knocked Back Flung
    Sucked In Standing, Crouching, and Air
    Sucked in Reversed Standing, Crouching, and Air

    and then there's the blocking junctions

    Guard Weak Standing, Crouching, and Air
    Guard Middle Standing, Crouching, and Air
    Guard Strong Standing, Crouching, and Air
    Guard Sucked In Standing, Crouching, and Air
    Guard Sucked In Reverse Standing, Crouching and Air

    Seems pretty daunting, doesn't it?  At first, yes.  But you don't necessarily need all of those hit junctions.  These are a universal list I use to start each of my projects, and remove them as necessary. 

    Now, in order to add these, click on your Game System tab, then click Hit Junctions.  You should see the original list I mentioned at the top.  You can rename these as I did, or leave them alone.

    Now, to add a hit junction, right click on the list and a small options window should pop up.  Click on Add > Hit Junction and you will then be prompted to name it.  Do this repeatedly until you have every hit junction you think will be necessary for your project.

    Here is the list shown in image form to make it easier to grasp.



    Yes, that is definitely a lot of hit junctions, but don't worry, in the long run, it saves you just that much more work.

    As you can probably tell, each of the hit junctions have been named for the property of attack, and the state the character was in when it got hit. They're pretty self explanatory, but for sake of user friendliness the properties of my hit junction list are as follows.

    Hit - is for physical bludgeoning attacks like those with fists and feet, the only exception is when it is followed by the words "Special Attack" or "Super Attack" these are universal hit junctions for most special attacks that don't cause an added effect (fire, electricity, etc.)  Hit Crouched Attack means that the character has been struck while standing by an opponents crouching attack, typically a weak kick or any variety of punch.

    Slashed - is for characters who scratch or slice, either with bladed weapons, a'la Vega/Balrog from the Street Fighter series, or with their hands, a'la Iori, or K9999 from the King of Fighters series.  Typically these are just the "Hit" hit junction animations used with a different hit spark.

    Face Plant - is used when a character's feet are knocked out from under them, usually by some form of sweeping attack that knocks them forward.

    Lit On Fire - is used when a character's clothing/body are ignited after an attack.  Ground means the character slides back while on fire, but does not fall over, Air means the character is knocked back slightly while on fire, generally as a result of being hit while jumping.  Launched means the character is either sent backwards or straight up while on fire, typically it is straight up.

    Knocked Down - means a character has been knocked off their feet.  Light, medium and heavy are all varying degrees of falling distance and height.  For example, a character being hit by low sweep kick would use Knocked Down Light, a character being hit by a standard uppercut would use Knocked Down Heavy, and a character hit by a Shoryuu Ken/Dragon Punch would use Knocked Down Special Attack.  A lot of these can also be used for throws too, but it seriously depends on the throw.  If you can't use these specifically for throws, you can set up hit junctions later to do so.

    Knocked Back Flung - means a character has taken a hit that knocks them either straight back at a wall, or a very long distance backward.  It's usually a crowd control tactic to get aggressive players off of you.  A good example is Chun-Li's Sou Hakkei palm thrust from Street Fighter Alpha 3 (X-ism only.)

    Sucked In - means the character is drawn in toward an attack.  The most famous example of which is Ryu's Shinkuu Tatsumaki Super Combo from the Street Fighter Alpha games, he jumps up and starts to spin, and if his leg clips the other player, they are drawn in for extra hits.

    Sucked In Reverse - is a followup to Sucked In, this junction turns the character around, allowing for them to be pulled back the other direction so the attack will function correctly.

    Guard - is just that, guarding.  It is used to create a "Blocked" junction.  An animation that results from blocking an attack.

    After I've set up my list, I typically go into my character that I'm editing and delete the "Slashed" skills, as well as the "Hit By Special/Super Attack" skills.  The reason for this is you can use the basic "Hit" animations to achieve the same effect.
    It's mostly a memory saver, so you don't have to if you don't want to.

    There are definitely more uses for Hit Junctions, but I've pretty much given you the universal standard of most fighting games.  

    Next, on to Common Images.  Common images are images shared by each character in a fighting game. These are essentially the same as Hit Junctions, but they can be used for a number of things.  I typically use common images during throws.  

    You don't really need that many of them, I usually only need about six or seven.

    Creating Your Fighter - Applying Collision Detection


    Collision detection is a very important thing.  This is what allows your game to function as more than just moving pictures on a computer screen.  The act of placing collision blocks around your characters has been streamlined tremendously over previous iterations of the program, and are useable via a simple skill object, rather than applying conditions directly to the animation itself.

    Right click on a skill, and the dropdown options menu will appear.  There are two collision options you have available.

    Add > Defense Frame - Used for collision involving taking hits, stopping movement, etc.

    Add > Attack Frame - Used for hit detection and application of Hit Junctions. 

    After clicking on either of these, one of two boxes will appear on your skill bar.  An FD box (), signifying a defense frame, or an FA box (), signifying an attack frame.

    If there isn't an FA box in a particular skill, it will always come preceded by an R box (), signifying a Reaction.  This box is where you place your Hit Junctions.

    First off, I will explain the types of collision detection available to use.

    To set the property of a defense frame, click on any combination of the three property boxes to the left of the editor window.  



    M. Number - refers to layering occupancy.  Any two defense or attack frames with the same value will cancel the one before it out, allowing you to move frames within an animation.  However, if the numbers don't match, you can have more than one collision frame in an animation simultaneously.

    Ratio - Acts as a damage percentage.  100 is full damage, 0 is no damage, etc.  So if you have a character with some type of super armor special technique or the like, you can allow them to take less damage while it is active.

    The four numbers beneath are X and Y values.  The top two refer to frame positioning, while the bottom two refer to frame size.

    No Property - This defense frame has no properties associated with it.  It is basically a placeholder frame.

    Striking - This defense frame creates the effect of your characters being solid. It is used when you want your characters not to be able to pass through each other, yet when struck by an attack, take no damage.

    Doing - This defense frame allows for hit detection only.  When a defense frame has only the Doing property, it can be passed through, but still take damage when struck.

    Throwing - Thanks to Proof and Acid I know what this is for.  It's used in conjunction with the [DS] - Detect Skill function "While Do Throw".  This enables you to decide what skills you want the character to be throwable in.

    Striking/Doing - A combination of the Striking and Doing defense frames.  It has the properties of both.  This will be your most widely used defense frame as it allows damage to be taken, but characters not to pass through each other.

    Striking/Throwing - A combination of the Striking and Throwing defense frames.

    Doing/Throwing - A combination of the Doing and Throwing defense frames.

    Striking/Doing/Throwing - A combination of all three defense frames.  A number of creators use this as a safe out when creating their characters because it's basically a universal hit detection frame.  I personally don't see much use for it that Striking/Doing can't accomplish.

    Attack - This is the universal attack frame.  This frame is placed around anything that is intended to cause damage to the other fighter, fists, feet, weapons, fire, what have you.

    Now that that nonsense is over with, we can move on to application of collision.

    Creating Your Fighter - Creating Basic Attacks


    Since this is a fighting game, we need attacks.  This section will teach you all about how to take a basic animation, and allow it to be used as an attack in your game.  This will cover punches, kicks, and throws.

    First, add your skills for your attacks to your skills list if you haven't already.  Apply collision detection as necessary, and then click on the frame that you wish to deal damage.

    Right Click and Add > Attack Frame to this image.  Click on the FA box that appeared behind the image you selected, and drag a box around the appendage necessary.  A red box should appear.



    This box tells the program that the selected appendage is what causes damage to other fighters.

    At the left side of the editor window are your attack properties. 



    This is a list of options available to you for editing aspects of your attacks.  

    M. Number - is identical to the option for Defense Frames, it allows you to negate or overlap defense frames within the same animation.

    Power - is used to determine the strength of an attack, the value set is equal to the number of hit points removed from the opponents energy bar.

    Cancel - allows an attack to chain into another attack after it hits.

    No Detection - causes the attack not to be detected by the Detect Skill condition fork.

    Cont. Hits - allows for the same attack to hit multiple times.  Typically after an attack lands, the rest of the collision is nullified so the attack will hit only once.  With this box checked, each Attack frame within the skill will count toward damage.  This is particularly useful for multi-hit projectiles and super attacks.

    No Sky Decision - means the attack will not affect characters who are in the air.  This is a good thing to use for throws and things of that manner.

    Guard Fail - renders the associated attack unblockable. Use with extreme caution as this can unbalance a character.

    While Guard - contrary to the Guard Fail property, this attack only works against guarding opponents.

    While Receiving - this acts as a sort of counter attack system.  Attacks with this property are only effective if you've taken a hit before using them.

    Shave - Attacks with this property still do a small amount of damage when blocked.  This property is typically used for special attacks.  The damage done by the attack when blocked is equal to 1/8 of it's original strength.

    The four numbers at the bottom are X and Y values and work exactly the same way as they do with Defense Frames.  The top two are positioning, while the bottom two are size.

    Good sound design makes a fighting game.  When you land a hit on someone you want to hear that satisfying "thwack" that tells you your attack connected.  You want to hear the loud thud when a character hits the pavement.  It's all about developing an atmosphere.

    Sound Select - is a simple dropdown menu that lets you select from the various sounds you've already uploaded into your project's database.  You'll notice ------None------- in the dropdown box because we have not uploaded any sounds, nor have we selected any to go with the animation. 
    (NOTE: It is very important that if you do not intend to have a sound, that you do not have a sound box in your animation, because if you do, the ------None------- in the sound properties will make your entire game go silent.)

    Loop to Point - This checkbox allows you to loop a sound.  Loop to point actually means loop to start, as there is no way to set a "point" for the sound to go back to.

    CDDA Track - tells the editor what track to load from a CD in your CD-Player.  This function only operates if you choose CDDA as your sound type.

    The Test section is pretty obvious:

    Play - allows you to play the currently selected sound from the dropdown bar.

    Stop - stops the sound while it's playing if you've selected to test it.

    The Add File section doesn't need much explanation either, however, for sake of a user friendly tutorial, I will do so anyway.

    WAVE - this actually refers to the .WAV format of sound file for the Windows operating system.  It simply means "Upload recorded sound effect in .wav format".

    MIDI - refers to synthesized audio generally created with a midi sequencer.  This is usually used for music, but due to the generally low quality, I stick to .WAV audio.

    CDDA - This lets you play a track from a music CD.  The premise is somewhat cool, if lacking in functionality.  The reason being, that it will load a track from a CD currently in the drive, not store the track for future use.

    Delete - should be obvious.  This button allows you to delete the currently selected sound file from the dropdown bar.

    Anyway, since we're loading a sound effect for an attack (at least I am, anyway), click on the WAVE button in the Add File section.  This will open up a folder and allow you to browse your computer for sound effect files you wish to upload.

    Select your sound and press Open.  You will then be prompted to name your sound effect (or music if you clicked MIDI)



    By default it will have the name of your file as the name in the prompt, which if you named them all accordingly shouldn't be a problem.  However, feel free to rename the sound to anything you wish and press OK.

    Select your sound (if the editor hasn't done so already) in the dropdown bar and then go ahead and test your animation.  The sound effect should play while the test is running.

    Now, since the sound itself is already loaded into the database, you can use the same sound in your editor multiple times without having to repeatedly load the file, easy huh?
    Now wash, rinse and repeat with all the sound effects you plan on using in your animations!

    Finishing Touches - Adding Objects to Animations


    We're in the home stretch now, folks!!

    Objects can be used for all sorts of things, from projectile attacks, to special effects on various skills.  Heck, I've used them to allow a fighter to summon his dog!  It's all up to you how you plan to use them.

    I'll be adding the energy flare to Rockman's Rock Upper special attack.  This somewhat differentiates it from being a basic Shoryuu Ken.  First, if you've organized your skills list like I went over at the beginning of this tutorial, go to the section you entitled as - Objects - and move down one skill.  Right click and select Add > New Skill and name the object you wish to create.  From here, create the animation for this object as you would for any other skill.

    Now here's how we put them into play.

    Go to the skill you wish to add the object to.  Right click on the frame that you want the object to appear on and once again, you see the familiar dropdown menu.  Click on Add > Object and a box with an O () will appear in your skill bar. 

    Once more at the left of your editor (and by now I bet you're either really tired of this part, or have figured this stuff out enough that you don't need this explanation, lol) is your Object Properties Utility.



    Position X, Y - is self explanatory.  These are the X (horizontal) and Y (vertical) coordinates for your object.  This tells them where to appear during your skill.

    Object - This dropdown bar is where you select the animation to be used as your object.

    Parent - This checkbox means the object will follow the object that created it.  Useful for things such as the fire on Ken's Shoryuu Ken or the Flare on Rockman's Rock Upper (which is the example I am using).

    UnCond - Or Unconditional, means the object cannot be removed via layering with a blank object, or by hitting the floor.  This object stays on screen until the animation is finished, or until another event (usually determined in the object's skill by a DS > Detect Skill command) occurs.  This is very useful for fireball type objects that you don't want to disappear for no reason while on screen.  Use this property wisely however, as you'll want to make absolutely sure you don't want to be able to cancel out the object in question.

    Shadow - This checkbox simply means that the object created will have a shadow  underneath it.

    Pic X Y - Is a bit of a misnomer.  This should probably say Screen X, Y, but that's neither here nor there.  This checkbox allows you to place an object relative to the play area, rather than relative to your character.

    M. Number - I'm still not sure what exactly the "M" means, but this is used for object layering.  You can set the number from zero to nine.  This simply tells the editor not to destroy one object if another object (with a different M. Number) appears on screen.  However, if you create two objects with the same M. Number value, one will cancel the other out.

    The Depth sections simply control if the object is in front of or behind other objects.

    In - The object is created behind the player characters.

    Out - The object is created in front of the player characters.

    Point - Allows you use the Depth Ap option to specify object depth.

    Depth Ap - Lets you select specifically what the depth of the object created is relative to other objects (0 being the farthest back, 127 being farthest forward.)

    It's Out - Is another bit of a misnomer.  Technically this could be called End of Object Skill.  What this function does is tell the player character to go to a specific animation that you select via the dropdown bar after the object's animation ends.

    Anyway, set your object's properties the way you intend for the specific skill.  This is how mine ended up.



    As you can see, my object is "Rock Upper Flare", set to a depth of "Out" with a "Parent" property and no "It's Out" condition.  This means the object will follow Rockman, and will be in front of him.

    As for the X, Y coordinates, you can either set these via the two numbers, or you can just click and drag on the editor field to move the crosshair to the specific spot you wish the object to appear.

    Finishing Touches - Tweaking Character Information


    Now, this section is mostly here for completeness' sake, as most of the properties here are intended as window dressing.  A lot of it, however, does have to do with functionality as well.

    On your character, click the Basic tab.  You'll be presented with a window that looks like this.



    This is your basic properties for your character.

    Sex - Obviously refers to your characters gender. Select from Male, Female, Both, or None.  This isn't entirely necessary, as in most cases, no one will see this information. (Unless they downloaded the character from the internet, of course.)

    Age - Refers to how old a character is.  This isn't particularly necessary to place either, but I do it anyway.

    Neutral Guard - This checkbox is used for Autoguard situations a'la the Street Fighter Alpha/Zero series.  Typically, you want to leave this off, because if it's checked, the character will automatically block all attacks if standing still.

    Sky Guard - Could be translated as Air Guard, but that's just nitpicking.  With this box checked your characters will be able to block attacks while in the process of jumping.  Very useful, and in my opinion, an absolute necessity unless you're trying to make your game feel like Street Fighter II.

    Vs. Mode - This allows the character to be played in Versus Mode.  Leave this box unchecked for characters such as endgame bosses (i.e. ANYTHING from the King of Fighters series).  In most cases however, you will want this box checked.

    The Gauge section allows you to edit various properties for your characters Life and Super meters.

    Life Gauge Max - This lets you set the total number of "Hit Points" if you will, that your character has.  This can be set anywhere from 1 to 1,000.

    Special Stock Max - Lets you edit the maximum amount of super meter levels a character has.  This can be set from 0 to 9.

    Special Gauge Max - Lets you set the total number of points required to fill up a super gauge.  This can be set from 1 to 1,000 as well.

    Starting Stock - Refers to the number of full super gauges the character has at the beginning of the first round of a fight.  This can be set from 0 to 9 as well.

    The Life Rev. section is a bit of an odd one.  I can't figure out exactly why the names are translated the way they are, but it's not a big deal.

    Start Pos - Actually refers to a section on the life bar to which a damage buffer will be added after so much damage has been taken by the player.  A perfect example of something like this is Jin Saotome from Marvel Vs. Capcom.  When he's down to one fifth of his total life bar, he starts into a super armor mode and takes half damage.  The number at the side refers to a percentile, Zero meaning the buffer is unused, and 100 meaning a full life guage.  

    Correction - Refers to how effective the buffer mentioned previously actually is.  Zero basically takes all damage and reduces it by 99%, while 100 makes the buffer virtually ineffective.  Note, the percentile values are likely somewhat inaccurate, but not by much.

    Special Gauge Increase - This section controls how much is added to your special gauge when various conditions are met.

    When Attack Hits - Refers to the amount of points added to your special gauge when one of your character's attacks make contact.

    When Enemy Hits - Obviously refers to the amount added to your special gauge when you're hit by your enemy.

    S. Ratio - This stands for Shave Ratio (according to the notation anyway).  This value controls how much damage the enemy character will take when blocking attacks that shave damage (any attack with an attack frame that has the "Shave" box checked).  When this number is left at 0 the damage is reduced to 1/8 of it's original total, whereupon if the number is 100, even if blocked the attack will still deal full damage.

    Char. Rev. - According to the notation, this stands for Character Revision.  What this actually does however, is shave off a portion of damage after a specific number of hits in a combo land.  I'm not sure of the exact ratio, but it seems to work sooner the higher this value is set.

    Interv. - Or "Interval" determines what is considered "Close" by the program.  I typically set this to somewhere around 90 which pretty much means the characters have to be damn near touching each other to be able to perform "close" attacks such as throws.

    Y Pos. of SideHP - This actually refers to the small lifebar that appears on your enemy if you click on the "Show Life" checkbox for your story mode setup.  This value sets the Y (vertical) coordinate for this small life bar.

    Guard Button - Remember Mortal Kombat?  Well that ain't what this does.  What this actually does is give you the ability to make a sort of single plane Super Smash Brothers type fighting system.  The reason for this is your character doesn't automatically turn around when changing sides.  You can face either direction at any given time.  The Guard button can be used like the barrier button from SSB/SSBM.

    That just about covers it for the basics tab.  More useful than you though, eh?

    Finishing Touches - Adding Artificial Intelligence


    This specific part of character creation has eluded many people, even since the days of FM95 has this feature caused trouble.  Why?  It's incredibly vague.  You set up some numbers and you're left with a big dark line and absolutely no clue what that line represents.  Well, I'm here to fix that for you.  Hopefully I can make it simple to understand using some of the AI data I assembled for CAPCOM - Mini Melee.

    To get started, click on the CPU tab for your character.  You'll be greeted with a hefty blank window with a white space at the left side.  The white space is the area for all your CPU commands.  Right click it and you'll be prompted with a small menu to either Add a new CPU Pattern, Rename an old CPU pattern, or Delete a CPU pattern.  Click on the Add a new CPU Pattern option.

    Just like with your players and skills, you'll be prompted to name it.  Simply go with what you're planning on telling the computer player to do.

    After naming the CPU Pattern, you'll see at the side that another properties area has appeared.



    This would be your CPU Pattern Properties utility.

    The Cond (Condition) area controls what conditions must be met to trigger the CPU to act.

    Prob - Or "Probability" is a percentile representation of how often a computer character will perform this specific action.  This value can be set between 0 and 100 (reflecting 0% and 100% probability)

    Character Is In The Air - This is a bit misleading.  By "Character" it refers to the CPU player.  This means that in order for the CPU to perform the action, it's character must be in the air.

    Enemy Is In The Air - This means that the player not controlled by the CPU must be in the air for the action to be performed.  This is typically a good one to use for jumping attacks, or anti-air maneuvers such as the Shoryuu Ken.

    The Interv. (Interval) section tells the CPU within what distance to perform the actions in the Pattern.

    Empt.  - Refers to the start of the covered distance the action will be performed (0 meaning right next to the opposing player, with the gap increasing the higher the value is set.)

    Inside - Refers to the maximum range past the Empt. value the action will be performed.  Note, the Inside value MUST be a higher number than the Empt. value. 

    You'll notice that when both the Empt. and Inside values are set, they produce a thick black line in the center window.  This gives you an idea of distance cover relative to the screen (scaled down, of course) that the character will perform the action.  You can use this to set up multiple versions of the same attacks but execute them from different distances from the opposing player.

    The Sk (skill) section is where you set the specific action you would like the CPU to perform.  You can set up combos by placing multiple attacks in, or you can tell them to move, and for how long by simply adjusting values and directional inputs similar to the way you created commands for your attacks.

    This is a snippet I've created for Dan from Mini Melee.



    This shows that Dan has a 26% chance of throwing a crouching fierce punch while in the vicinity of 0 to 130 from the opposing player.  The Down arrow in the Sk section makes him crouch, while the Fierce Punch skill selected makes him attack.

    Here's what the center window looks like.  I can't show you the actual CPU pattern names, as they'd give away Dan's programming, but I can show you what your finished AI might look like



    As you can see, he's got a number of moves that he'll only perform up close (the closer to the left the end of the black bar is, the less amount of distance apart the characters will need to be.)
    You'll notice also, that some of his CPU patterns will be executed from anywhere on screen (the extra long black line.  The "Inside" value was set much higher than the "Empt." value, setting a large cover distance for the CPU pattern.)

    Creating Stages - Creating Scrolling


    Alright, now for the finishing touches to stage creation.  Now that we've got all our layers set up, we'll learn how to create a parallax scrolling effect.  Go to the first layer on your stage (in my case, Wily's Factory), and click on the name. 

    At the left side of your editor window, the Layer Properties utility should now be displayed.



    You don't have many options, but they're all pretty important.

    L R Connect - This checkbox repeatedly tiles your background layer horizontally.  This is typically used for stationary backdrops such as sky.

    U D Connect - This checkbox tiles your background layer vertically, more often then not, you'll pair this with L R Connect.

    The Scroll section is really what we're looking for, however.  

    Width - This checkbox allows you to set the horizontal scrolling for your stage.  The number at the side will control how fast the stage scrolls sideways.

    Length - This checkbox allows you to set the vertical scrolling for your stage.  This makes the stage move up and down when your characters jump.  The number at the side once again controls how fast the scrolling will occur.

    Now, check both the Width and Length checkboxes.  On this layer, set both values to 100.  This is the optimum setting for your basic layer, as it will scroll at the speed your character moves.

    You'll notice the bounding box around your stage frame in your editor got larger.  This is why I mentioned earlier not to move the frames.  Using the bounding box as the entire border for your stage, you can align your layers so they display correctly on screen.   With the layer I'm currently on, I'm going to align the bottom of the image with the top of the bottom line of the box, this will keep the floor where I need it to be.  You can also move the image horizontally if you need to center your stage in the play window.

    Now, in Rockman's stage, unfortunate to say, there isn't any parallax scrolling, as it's just one big layer on top of some backdrop effects (smokestacks, what have you.).  But that doesn't stop me from explaining how to create the effect in your own stages.

    Following the same procedure you used to create your initial scrolling, click the check boxes for the next layers back.  Now, you'll need to reduce the Width and Length values accordingly, as land farther back on the horizon will scroll more slowly than anything toward the front.  The amount you need to decrease is entirely dependent on how many layers are in your stage.  If you've got a typical stage, you'll have around 4 layers (the ground, the mid horizon, the far horizon, and the sky).  If this is the case, you'll set the mid horizon's Width to around 35-40 and the Length to around 50-65 (Note, this is also VERY dependent on how high your characters jump, so some testing will be in order.)  For the next layer back, the far horizon, set the Width to around 15-18, and the Length to around 35-45 (once again, judging for jump height).  Now, if your last layer is the sky, just leave it alone.  A stationary sky produces the effect we're looking for.

    I know it seems a bit complicated, but with some tweaking, you'll figure it out and be producing rolling landscapes in no time!

    2D Fightermaker 2nd - Creating Demos


    This is more of a subsection than anything, so I will keep it short.  The reason is, if you've paid attention while creating characters and stages, you'll have no problem whatsoever creating demos because the process combines both.

    What are demos?  Well, it depends.  They're used for a number of things.  The exact list however can be found in the Game System, Basic tab of your project.

    The barebones uses for demos are

    Title - This demo is your title screen.

    Character Selective 1P - This is your character select screen for single player modes (call it "Arcade" if you have no plot, or "Story" if you do.  This isn't totally necessary, but it's nice to let people know if you've got a plot in your game.  Call it whatever you wish.)

    Character Selective Vs. Single - This is your character select screen for your basic one on one versus mode.

    Character Selective Vs. Team - This is your character select screen for Team Vs. Mode (if you choose to include one.)

    Game Over - This is actually a demo used for a Continue screen.

    Opening Demo - This demo is used for an opening cutscene for your game.  This is often called an "Attract Mode" screen by those in the arcade game business, because all the flashy imagery was intended to attract players to put quarters in the machine.  You can use this to show off graphics, explain game story, whatever you want.

    That's the basic list for your demos.  However, you can also use them for in-game cutscenes, credits sequences, and things of that nature that can be used during story mode.

    One thing that's a bit strange to most players is creating a character select demo.  I'll explain the steps to do so here..

    First,  your character select screen backdrop and import it to the editor as a demo.  Place this in any of the Character Selective dropdown bars in your Game System, Basic tab.  

    Now, you may notice a "Char. Selection Picture" tab at the far right.  Click this tab.



    You will be presented with the following editable values:

    Character Start Position X, Y - This tells your game what position to place the first portrait on your character select screen.  X being horizontally, and Y, being vertically.

    Distance Between Character X, X - This is a typo.  It should be X, Y.  This tells your game how far apart to place each portrait on your character select screen, X being horizontally, and Y being vertically.

    Columns and Rows in Select X, Y - This tells your game how many portraits to place in each row (from 1 to 50. 50 being the maximum number of characters your game can have) and how many columns of portraits there are (also from 1 to 50.)

    Player 1 Cursor Position X, Y - This tells your game where Player 1's character select cursor starts, with the X value being the horizontal coordinate, and Y being the vertical.

    Player 2 Cursor Position X, Y - This tells your game where Player 2's select cursor starts.

    Player 1/Player 2 Selection Width X, Y - I'm unsure as to what these actually do.  I've played with the values a number of times, but they haven't done anything.  I would assume they tell teh cursor how far to jump each time a direction is pressed.  I will update this entry if I figure anything more out.

    Advanced Techniques - Variables Explained




    Here's something I know some people have trouble with.  It's not really as hard as it seems, however.
    Now, this section isn't a demonstration, as much as it is a definition of what variables are, and what they can do.

    Variables are an incredibly useful tool.  They're the most versatile thing in the world that Enterbrain chose to add to their program, and I'm a happier person for it!

    Anyway, on to the explanation.  Variables are just that... Variable.  They are numerical values that can be altered at any time during a skill, and that variable can then be used to tell a skill to do something else.  Variables come in three different flavors

    System Variables - These variables are system wide.  They affect not only the player who activated them, but also anything else programmed to use that same system variable.  These are typically used for instances involving whole stages (music changes when time's running out, music gets faster when health is low, etc.).  These are not their only uses of course, they're only a few examples.  

    Character Variables - Affect one character.  They remain constant until one of that character's skills tells them to change.  They do not affect anything involving the stage, or other characters.  these are typically used for things like Transforming characters.

    Task Variables - It was my original belief that these affected only the skill they are present in.  This is not the case.  I have tested this theory.  The difference between these variables and character variables is not a big one, but it is important if you want to use variables correctly.  

    Task variables function like character variables on one character OR object, not both.  For example, let's say you had a projectile that altered a character variable.  This character variable can be accessed by any skill used by your character and still function properly.  If you tried this with a task variable, the variable changed by the object would not affect the character at all.  

    To place a variable in a skill, simply right click the square you wish to place it.  The dropdown menu will appear as usual

    Scroll down to Add > Condition Fork, and select [V] Variable. A square with a V will appear in your skill bar.  Congratulations, you've added a variable condition to your skill.

    With the Variable square highlighted, at the left side of the editor window, you'll have your Variable Properties section.



    The options you have here are:

    Var - This section allows you to pick a specific variable from the dropdown bar (Task Var A through P, Char Var A through P, and System Var A through P).  You have a lot of variables to work with, and this is a good thing.

    # Calculate - This section is where you edit the variable specifically.  The numerical value is what you use to set the variable to a value you wish to use later.

    No Calculator - This is the default setting.  This means that no change to the variable's numerical value will occur, and you'll notice the number space is blacked out accordingly.

    Replace - This means that the numerical value the variable currently has set (default 0) will be replaced with whatever value placed here.  (I.E. Char Variable B is currently 4.  If you set a Replace condition with the numerical value as 1, the new value of the variable will change from 4 to the new setting of 1.)

    Add - This will add any number (positive OR negative) you set in the value area to the current total of the variable.(I.E., Char Variable B is currently 4, if you set an Add condition with the value of 1, it will add 1 to the value, making the new variable value 5.)

    Use This - When this box is checked, you can select a number of options from the dropdown bar below.

    The dropdown bar has a number of options for you to use

    Task, Char, System Var A through P - These settings allow you to use the current value of any variable on the list.

    Data X/Y Coord - These settings allow a variable to use the current X or Y coordinate of an object at the time the variable was accessed.

    Data Map X/Y Coord - These settings will allow a variable to use the number relative to the X or Y coordinate of an object relative to it's current place in a stage.

    Parent X/Y Coord - These settings allow an object's variable to use the number relative to the current X or Y coordinate of the parent object that created it.

    Data Time - This allows a variable to use the current time as a numerical value.  This can be used when you want the timer to activate certain conditions.

    Data # of Rounds - This allows a variable to use the number of the current round as a value.

    Now we move on to the important stuff.  This is how to make a variable work.  The Condition Branch section will read a value from a variable based on the options you've picked, and use the condition you set here to create a result.  The functions of this section are as follows:

    No Cond Branch - This means that there is no condition branch set.  This is used when you want only to change a variable's value, but not have anything besides that occur.

    It's The Same - This means that when the variable value is equal to the number specified in the Condition Branch section, the variable is found TRUE and will then go to the specified area in the skill as determined by the dropdown bar.

    It's Above - This means that when the variable value is higher than the number specified in the Condition Branch section, the variable is found TRUE and will then go to the specified area in the skill as determined by the dropdown bar.

    It's Below - This means that when the variable value is lower than the number specified in the Condition Branch section, the variable is found TRUE and will then go to the specified area in the skill as determined by the dropdown bar.

    Advanced Techniques - Creating Working Projectiles


    Now I hate to say it, but even projectile attacks don't work as easily as they should.  They pass through each other, nullify one, but not the other, and other horrid things like that.  So here's something I came up with to solve that problem.  

    I drew the inspiration from Proof58's Akuma character.  I was having problems creating a fireball that worked, so I looked through as many resources as I could find.  Akuma had something that ALMOST worked.  He had an extra attack frame below the ground that acted as collision for the projectile.  This is how I fixed the method.

    First, create your projectile.  The animation should include both the projectile and the destroy frames for the projectile





    Put your move icon at the start, set the speed to whatever you need.  I set mine to +4.80.  This is just slightly faster than Rockman moves.

    Now, what you want to do is put an [SG] GoTo condition that points back to the beginning of the skill in the frame before the destroyed animation.



    Set up your attack frames as normal and set up your hit junctions.  

    Now for the strange part.  You're going to create an attack frame just to the front of and WAAAAY below your projectile.  Give it an M.Number of 1 so that it doesn't cancel out your actual attack frame.  You're also going to need to give it an attack strength so that the effect will work later, otherwise projectiles will behave unpredictably.  I give it the same strength as the actual attack just for ease.



    Now, afterward, create a Defense frame at EXACTLY the same place as the attack frame you just made.  Give it the Doing property.



    Strange, I know, but it works!  Now, create two DS boxes (Place these at the beginning of your animation).  Set one of them to Attack Hits, and the other to Defending.  You'll have them both point to the Destroyed part of your animation.





    Now, you don't want the projectile to keep moving when it's destroyed, so place a move box at the beginning of the destroyed animation, set to Replace and all values at 0.00.You may also have to set your DS pointers again, check those and correct them if necessary.  It would also be a good idea to add an FD and FA box after the move box to neutralize the attack boxes around the actual projectile.

    Last but not least, right click on the last box and select Add > End Skill.  This will end the destroyed skill and get rid of the object after it's destroyed.  Add the projectile object to the necessary skill.  Now, test it out.  Have both characters fire at each other.

    Advanced Techniques - Single On-Screen Projectile (Hard Type)


    Don't say I didn't warn you!! This caused me anguish for DAYS.  But, wanting to bring as many techniques to the table as I can, I will explain to you in detail how I accomplished this feat.  This is seriously a lot of effort for very little product, so use at your own risk!  Also, this will be a LONG explanation.

    (NOTE: This technique is fairly sloppy as it was put together in a cloud of frustration.  I'm sure there's a neater way to do this.)

    What this technique does is allow a character's shot animation to end, and let the character move as normal, but not be able to fire a second projectile until the first one collides with the enemy, another shot, or leaves the screen.

    The reason I had to come up with this method in the first place is that the easier method didn't work for this character.  This is likely a result of how Rockman's attack is set up (having to use variables to check whether he's standing, walking, what frame of walking, crouching, what have you).

    Now, since Rockman's fierce punch is a projectile, there were a number of things I needed to do to make this work the way it does in Marvel Vs Capcom. 

    When you press fierce punch, Rockman fires a buster shot.  Yet if you press Fierce punch again  He doesn't react at all, no animation changes.  So what I had to do was set up a variable to determine what Rockman was doing.  I used character variable M.

    M = 0 - Rockman is Stationary, either standing or crouching.

    M = 1-12 - Each individual frame of Rockman Walking forward.

    M = 13-24 - Each individual frame of Rockman Walking backward.

    As you can probably guess, I had each animation change this value accordingly (the start of his crouching and standing animations set Variable M to 0, as do any of his jumping animations, while there's a variable change between each frame of his walking that sets them accordingly.)  I also set a Detect Command input after each variable change to make sure that if you stopped pressing forward or backward, it would stop the walking animation.





    So on and so forth.  

    I created a skill called "Shot Fail" which is what is used when Rockman is stationary and attempting to fire a second time.  It's got a DB at the start set to Crouching, Formed, that points to the second part of the animation (which is Rockman crouching.)  Both images in this frame have a wait time of 1.  The two M boxes make sure his movement stops if needed, and I placed an End skill at the end of each of them to tell the skill to stop when I need it to.



    The FD objects in the animations are identical to what they would be if Rockman were standing or crouching.

    I then created a skill called "Shot Variables" that holds each possible outcome for the Character Variable M and tells it where to point.



    Now, I had an object created during Rockman's shooting animations that set Character Variable P to 1, this was the variable that tells him not to be able to shoot more than one object at a time.  I also added an End Skill followed by an [SC] Call Skill box that points to the Shot Variables frame.



    At the start of the animation I placed a variable check that says "If Variable P is more than zero, jump to SC - Shot Variables".  I did this for both his crouching and standing versions of this attack.

    What this will do is cause Rockman not to be able to use his fierce punch while the shot is on the screen, because it will constantly call a shot failure.

    This however is only half the battle.  What we need to do now, is get the variable to reset itself when the object is no longer in play.  This is what caused my torment.

    First, go to your projectile animation, at the beginning of the destroyed animation, place a Variable that resets Character Variable P to zero when the object is destroyed.



    Now, here's the complicated part.  There is NO exact way to do this, as it all depends on the speed of your projectile.  I placed a variable setup between each frame of the buster shot that sets Character Variable C to 2.  



    Once again, seems strange, yes, but it's VERY necessary.  I then created an object called "Variable Object" which would then be used to deal with Character Variable C.  I placed a blank image with a wait time of 3, followed by a variable edit.  This variable edit would constantly subtract 1 from Character Variable C.  I placed an SG after this to have the object keep looping and therefore keep adjusting the variable.  After the SG, I created a two variable objects.  The first one reset Character variable P to zero with a Condition fork stating that whenever Variable P equals 1 to jump back to the beginning of the skill to start the whole process over again.



    Now, have this object created during your shot animation with an M.Number of 9 and a "Parent" property set so it will follow your character.



    What this object does is subtract from Variable C so that when the object is no longer on screen it'll tell each variable to reset as intended.

    That's how it's done, and let's never speak of it again.

    Advanced Techniques - Floating Nameplates


    I do have to admit, I'm probably not the first one to have come up with this technique.  This however, is the method I figured out on my own while working on CAPCOM - Mini Melee.

    First, go into your character, and right below your "Stage Face Pic." skill, create a new skill called "Stage Name".  This of course is up to you, but it'll help you keep things organized.

    Upload your character's name (preferably using a private palette) into this skill.



    Now, in the image properties utility check the "Ignore the Direction" checkbox.  This will keep the name from flipping backward on the second player's side.  



    You can center the name on the white crosshair if you want, in fact, it's a decent idea because it'll help you place it better later on.



    After you've got that done, create a Goto Skill condition at the end of the skill that tells it to go back to "Stage Name 1".  This will keep the name on screen.



    Now, go to your Stage Face Pic. skill and put an Object square at the beginning to create your nameplate.  Select the Stage Name skill in the dropdown bar and give it the following properties.

    Advanced Techniques - Creating Super Combos


    Super combos... Where would today's fighting games be without them?  

    First, a little history lesson... 
    (Skip it if you want to, but I feel like it's necessary to explore the roots of certain game related trivia.)

    Most people would say Capcom started this trend with Super Street Fighter II Turbo, this is not the case however.  SNK Actually did it first.  Capcom released SSFII Turbo in 1994 on it's CPS2 hardware, but two years before, in 1992, the Neo·Geo housed a game known as Art of Fighting.  This game was SNK's answer to the popularity of the numerous iterations of the Street Fighter II game, and SNK was in fact, targeting Street Fighter's Ryu in it's main character of Ryo Sakazaki.  Capcom of course, retaliated in kind by creating Dan Hibiki a few years later.  
    In Art of Fighting, you could use a technique called a Super Death Blow which drained your character's stamina bar, which made your attacks fairly useless until you recharged your stamina. 

    Anyway, where SNK paved the way, Capcom later perfected.  With the release of the Street Fighter Alpha series of games, came a revamped and much more varied Super Combo system (Super Combo was the name given to these techniques by Capcom).

    Capcom introduced a system that no longer required a stamina bar, meaning your special moves still worked as intended, and the super gauge was multi-level.  You could store up to 3 at a time, and then release them using either 1, 2 or all 3 levels of your super gauge.  This allowed for a level of strategy, go with a weaker Super Combo, but save some for later, or blow it all in an all or nothing strike. 
    The system used by Fighter Maker is technically that of Street Fighter Alpha, though if you wish, you can tweak it to be much like Art of Fighting if you wanted to.  However, since what most people think of for super attacks is that of King of Fighters or Street Fighter Alpha, that's what we're sticking to.


    Here's how it's done.

    First, create your super attack animation. I've already done this, as I compiled Son Gokuu to be able to transform, so I needed each of his animations completed.  I will demonstrate using Gokuu's famous Kamehameha.



    As you can see, the animation is pretty much complete.  I've got my transformation variables, objects, special effects, what have you.  This is not yet a super combo however.  Creating these attacks is possible in one step.  At the beginning of the Super Combo animation right click and select Add > Gauge - [GS] Check Special Gauges Fork. Of course, once again, at the left side of the editor, you will see the properties editor.



    In the Setting section, we have

    Special Gauge 'X' From - This basically means Current Special Gauge level.  The number is used to determine how much special gauge you have stored.

    When Little - Should more accurately be translated as "If Below", meaning that with this option selected, if your special stock is BELOW the number specified in "Special Gauge from", the skill will continue.

    When Alot - Is also more accurately translated as "If Above", meaning that with this option selected, if your special stock is ABOVE the number specified above, the skill will continue normally.

    Add To Advance - This allows you to place a number to add (positive number) or subtract (negative number) in order to advance the skill as intended.

    If It Failed - This is used when the conditions you have set have not been met (for example "Special Gauge 2 from When Alot, meaning when gauge is over two levels, if you try to use the move with one level) the skill triggered instead is placed in this space.

    Now, Here's how to create a level 1 super combo.  In "Special Gauge from" leave the number as zero, then select the option "When Alot".  This tells the game that this skill will only work when the character's gauge is above zero, meaning one level or higher.

    After this, place -1 in "Add to Advance".  This will subtract one level from your super gauge when you use this skill.

    Now finally, in "If it failed" you can place any animation you wish.  Since Son Gokuu's super combos are both done with heavy punch, I'll tell it to go to his standing strong punch if the gauge is too low.



    And that's it.  Test Play your game and try it out.  If done correctly, it should behave as such.

    First, you'll notice Son Gokuu has 3 levels stored.  



    But, now, I'll use the Kamehameha.  



    Now, not only did player two take a nasty hit, but if you'll take a look at the bottom left corner, Son Gokuu's super gauge level has decreased.  This is exactly what we're looking for.  Also, after his super gauge is depleted completely, the command will cause him to punch instead of firing a beam.

    And that would be that.  You can change the values to allow for second and third level supers as well.  Just use these settings instead of the ones demonstrated before.

    2nd Level Super Combo - Special Gauge 1 from, When Alot, -2 Add to Advance

    3rd Level Super Combo - Special Gauge 2 from, When Alot, -3 Add to Advance.

    Simple, huh?  The skies the limit as far as levels.  You could make a super attack that takes 9 bars to use, but in most cases, the fight would be over long before it saw the light of day.

    Enjoy!

    Advanced Techniques - Creating Transforming Characters


    Transforming characters.  Generally these are the characters with the most potential to be cool or memorable in your fighting games.  The reason for this is that transforming is a gimmick.  When you see a character change into something else mid-fight, it sticks in your head.

    I remember back in the FM95 days that this was a bitch and a half to pull off... Not so much anymore since the introduction of variables!  They've made thing 100% easier.

    Now, the first thing you need when creating a transforming character is the base character itself.  Since Dragonball Z has ready access to characters who can transform, I will be using Son Gokuu from Dragonball Z - Super Butouden on the Super Famicom.



    Sure, not the greatest LOOKING choice of the three Super Famicom DBZ games, but it already had the resources I needed to make this process go faster.

    First, you're going to need to get all your animations set up as you would for a regular character.  This includes standing, movement, crouching, attacks, hit junctions, etc.  

    After that, it's time to get creative.  Create a skill that's used specifically for the transformation sequence. (I would normally make this skill a 3rd level super combo, so it's not overpowering).  Considering I'm using Son Gokuu, I'll call it "Super Saiyajin Transformation".

    Compile your animation and any special effects you want it to have, and then at the end, have it control one of your character variables. (I used Character Variable P).  Have it set this variable to whatever you wish, though more than on+1 is usually overkill.



    I had also set a variable condition at the beginning to be sure that when the move is used while transformed nothing will happen.  I'll go into that more in a bit.

    After you've set this skill up, you basically need to go back and alter all your other animations.  What I mean by this is you need to create your character's transformed animations.  So what you're basically doing is creating two characters in one.  Here's the altered standing animation so you can see what I mean.



    Basically, what I did was duplicate the original animation at the end of the original using new frames.  I separated the original and transformed animations with an End Skill condition at the end of the normal version animation/beginning of the transformed animation (dead center in the skill) so that the animations wouldn't both cycle.  Now, at the very beginning, I placed a Variable condition at the beginning to check whether or not the transformation variable has been activated.



    As you can see, the condition says "When Character Variable P is above 0 Goto Standing 5".  This tells the animation to skip straight to the Super Saiyajin standing instead of his normal standing whenever the variable is above zero.  Now, simply repeat this process with every animation in your character as well as setting variable conditions at the start to point to the correct areas of the animations.

    On another note, this same technique can be used to create a revert to normal animation.  Simply have a variable at the beginning of the transformation skill point to a revert animation if the character variable is above zero.

    That's pretty much it for transforming characters.  The beauty of the fact that you need to replicate each animation is that you also replicate the Attack Frames too, so you can actually make the transformed versions have stronger attacks than their normal forms!

    2D Fightermaker 2nd Round - Program Tabs/Icons Reference



    There are a lot of tabs in Fightermaker 2nd.  They're not entirely difficult, but many tabs use the same names, so this document and other documents/forum posts can get slightly confusing.  What this section is for is to show you what all the Program Tabs and various Icons throughout the program look like and how they are used/what they do.

    The format will have the name of the tab or icon in bold print and the definition in standard.  If it is a subcategory of a tab however, the original category tab will be placed next to it in italicized font encircled by parenthesis.

    We'll start out with the program tabs.



    _________________________________

    Game System - This tab is where you set up various aspects of your game, Basic settings, scripts, the master hit junctions list and common images, as well as settings for your character select screens will be found here.

    Basic (Game System) - This tab allows you to set up basic functionality for your game.  Pause time after collisions, your demos, what kind of modes your game will have available, whether or not to use the "offset" function, etc.  Not to be confused with teh Characters basic tab.

    Basic Script (Game System) - This tab is where you set up all your scripts for your game.  Lifebars, timer numbers, win markers, round animations, various decorations for your screen, it's all here.  

    Hit Junctions (Game System) -  This tab is the master hit junctions list.   This is where you set up the initial hit junctions that your game will use.  This is not to be confused with the Character hit junctions tab, described further down.

    Common Images (Game System) - This tab is the master common images list.  Similar to the hit junctions list, this is where you set up the initial common images your game will use.  Once again, this is not to be confused with the Character common images tab.

    Char. Selection Picture (Game System) - This tab is more accurately described as Character Select Setup.  What this tab allows you to do is set the starting position for the two cursors, the number of rows and columns, the number of characters, and the space between each portrait on the character select screen.
    _________________________________


    _________________________________

    Characters - This tab allows you to access various options for characters in your fighting game.  This is where you edit their animations, their skills, their collisions, their hit junctions, as well as attack inputs and artificial intelligence.

    Basic (Characters) - This tab allows you to enter basic data about your character.  Some of it is merely aesthetic, like age and gender, but others are for function, such as whether or not the character is useable in certain game modes, how much energy they have, special stock, etc.

    SK (Characters) - This tab is where your characters animations, also called "Skills" are edited.  It is here that you set animation frames, framespeed, as well as collision detection/reactions.  This is the focal point of character creation.  You'll spend most of your time in this editor.

    Hit Junction (Characters) - This tab is where you assign animations to the hit junctions created in the master list.  It's a simple set of dropdown bars associated to your various hit junctions.

    Common Images (Characters) - Similar to the Hit Junction tab, this is where you assign frames to each of the common images in your game's master list.  The difference is however, that this is treated much more like skill creation in that you have the frame displayed in an editor window, and that frame can be changed via numbers at the top, and positioned anywhere in the field.

    Commands (Characters) - This is where you place the control input commands for your characters attacks.  The area at the right side allows you to assign animations to character situations, and the area below it allows you to set the command.

    CPU (Characters) - This tab is where you set up your characters' artificial intelligence.  It can be a bit daunting at first, but when you get the hang of it, it's no big deal.

    Story Mode (Characters) - This tab lets you set up your "Story Mode".  You set up fights, in between demos, special circumstances for secret fights, endings, etc. 
    _________________________________


    _________________________________

    Stages - This tab is where you edit your game's stages.  This is handled much like character creation, though there are far less sections to worry about.

    Basic (Stages) - This tab lets you set up the music for your stage.  That's all.

    Script. (Stages) - This tab is where you design your actual stages.  This is a lot like the skill editor in Characters, but this time each skill layer acts as a stage layer, with those farthest at the top being the farthest back, and the bottom of the list being the closest.
    _________________________________


    _________________________________

    Demos - This is the tab that lets you edit various cutscenes and other more functionally adapted demos such as character select and continue screens. 

    Basic (Demos) - Pretty much identical to the Stages Basic tab.  This tab allows you to edit the music for the demo (whee!), set how long the demo actually is, and whether or not it can be skipped with a button press.

    Script. (Demos) - Once again, this is pretty much just like the Stages tab of the same name.  You design your demos here much the same way you edited your stages with the same layering rules applying. 
    _________________________________

    Now, we need to take a look at the icons in the game.  These only show up in the various editors of the program, meaning Basic Script, Stages, Demos, and Characters.

    These have already been discussed in various sections of the document, but they will be placed here to cut down on browsing time.

    Image Box - This icon represents an image in a skill or animation.  You'll see a lot of these while creating a game.

    Motion Box - This box is used when you want to set motion parameters for a skill. 

    Object Box - This box allows you to create an object in an animation using a skill from the character's list.  This is typically used to create projectile attacks or special effects.

    Sound Box - This box is used to play a sound during an animation.  This sound can be a .WAV file, a .MID file, or a CDDA Audio track (the track played depends on the current CD inserted in the drive).

    Frame - Attack Box - This box allows you to place a collision detection square for an attack within a frame of animation.  You can set the damage of the attack with this as well.

    Frame - Defense Box - This box allows you to place a collision detection square used for detecting if an attack lands within a frame of animation.  You set the collision properties type here as well.

    Reaction Box - This is used for hit junctions.  You assign a hit junction to each dropdown field in the reaction box to tell a character how to react when hit by this attack.  NOTE:  If the Power setting in your Attack Frame is zero, the hit junctions will not respond correctly, if at all.

    Detect Skill Box - This is a bit of an odd one.  This is used to detect various conditions within a skill, such as whether or not an attack lands, if the character is landing from a fall or jump, defending, etc.  Using these conditions you can tell a skill to divert to another skill or a different part of a skill depending on what happened.

    Cancel Condition Box - This is used to make a skill cancelable.  You can set various conditions that can let you stop a skill mid animation to divert to another skill.  This is useful in creating string combo systems such as the "Magic Series" combos from the Capcom Vs. games, or Post-MK3 Mortal Kombat titles.

    Basic Divergence Box - This is another odd one.  It's very similar to the Detect Skill box, except that it uses even more broad spectrum detection methods.  Instead of using a condition that is happening to a character, the Basic Divergence box uses conditions such as whether specific directions have been pressed, a character is standing, crouching, etc.  This can also be used as a branch.  If a condition isn't met, you can have something happen as well as if a condition is met. 

    Variable Box - This is by far my favorite and arguably the most useful new function introduced in Fightermaker 2nd.  What this does is allow you to set a numerical variable that can be accessed at any time to divert one part of an animation to another, or to a new skill entirely.  The skies the limit with these.  You can create Transforming characters, set specific functions to happen based on said variable, such as a character's animations changing depending on health level, etc.  Use your imagination, chances are you can do it if you use variables.

    Random Box - Think of this as a dice roll.  You can set this condition to load random animations depending on a numeric ratio.  These numbers are generated at random during the game.  This is ideally used to create multiple win poses or start animations.

    Detect Command Input Box - This is another very useful function. This allows you to set a condition that when a command is entered during a skill, the skill can continue, it can divert to another skill, or it can cancel the skill.  I typically use these for rapid tap moves, such as Chun-Li's Hyakuretsu Kyaku.

    GoTo Box - This simply tells a skill to go to another animation when it reaches the point where this function is.  It's typically used to loop animations an infinite number of times.  

    Call Skill Box - This will call a skill to activate in the middle of a currently active skill.  Not particularly useful, but it can be used during super moves, or character specific special attacks.

    For (Loop) Skill Box - This can call a specific section of an animation to loop a number of times as determined by the numeric input in the function properties area.  This is a fairly useful object when you don't want to waste a character variable to create a loop-then-stop type skill.  A typical sort of use would be something like Ken and Ryu's Tatsumaki Senpuu Kyaku special attack.

    Change Shape's Condition Box - This calls a common image for an opposing character during a character's skill.  This is typically used during grapples.

    Change Skill Box - This is similar to the Change Shape's Condition box, except this time, it calls a hit junction from an opposing character.  This is also used during throws, and is typically a bit more useful as it can be applied to many more situations.

    Check Special Guages Fork Box - This allows you to script a condition fork based on how many super stock a character  has stored.  This is mostly used for determining which attacks drain your special stock, i.e. Super Combos, and by how much.

    Check Life Guages Fork Box - This is identical to the Check Special Guages Fork box, except that it uses your character's life guage as a base instead of special stock.  This can be used to create a similar situation to the original Art of Fighting series, where as a character took more damage, the more beat up they looked.

    Change Guage Value Box - This allows you to change the current value of your/your enemies Life and Special Stock guages.  These could be used during various hit junctions (i.e. being thrown), or similar to a charging animation, a'la King of Fighters.

    Background Effects Box - This is more accurately labeled Special Effects, as it not only deals with the background, but can be applied to a character as well.  This allows you to set various special effects such as color fades or camera shakes to be activated during a skill.

    Color Box - This works similarly to the Background Effects box, in that you can apply a color modification to your character.  This can be as simple as a hue change, or as complex as a graduated fade (this is achieved by using multiple color functions in one skill).  You can also use the Alpha settings to make your character transparent.

    Time Stop Box - This allows you to stop time for either you, your enemy, or both of you.  This is typically used in combination with a Background Effects box during the beginning of a Super Combo to signal it's use.

    After Images Box - Neat? Definitely.  Practical? Um... sort of... This is used to create after images that linger after your character during a skill.  These can be used to achieve a look of a character moving at immense speeds, or just as a flashy overall effect during skills.  A good example is Johnny Cage's Shadow Kick from Mortal Kombat, or the Variable ISM Custom Super Combos from Street Fighter Alpha 2 and 3.  Why are they impractical?  Well, mostly because they require you to place a condition that will trigger the after images to turn off during all your hit junctions as well as at the end of a skill that uses them so that you won't be stuck with afterimages on every skill after the one that triggered it.  The effect is worth it though.

    End Skill Box - This is used to tell a skill to end immediately upon reaching this condition.  This is typically used for variable related skills such as a transformed character (where you have more than one animation compiled on the same line, but don't want both of them active).  You typically don't have much use for it otherwise.

    End Command Box - This box means that this is the end of the command.  Simply place this on the last line of your command.

    More Arrow Box - This box means that the command continues on until it reaches the .  To place this in your commands, simply click on the first and it will appear in it's place.

    Free - This is a freebie space in the command.  If you place these between each line of your command you can make your game a little more keyboard friendly.

    Wait Dot - This is a call back to the Tekken/Virtua Fighter series of games and simply means leave some space between one part of the command and the next.

    Input Arrows - These are the input arrows for your commands. Each of them responds to the direction needed to be pressed during a command.  Click on the a few times to scroll through each of your arrows and set your commands the way you'd like.

    Roll Arrows - These arrows are a bit of help from the people at Enterbrain to it's program's users.  Instead of having to place 3 separate arrows to create a "Roll from 'this' to 'this'" command, you simply need to place one of these between the two corresponding lines.  These are accessed in the same way your command arrows are. Simply click a few more times.

    Input Buttons - These are the buttons you use for your punches and kicks, but that's not all they're restricted to.  You can have each button do something different, it's all up to you.  Obviously the six-button setup is intended for those who wish to create a CAPCOM style brawler, but you can use as many or as few of the buttons as you like.  Simply click on the button to make it active (they will look like the highlighted buttons in the example.)

    Command Differentials ,, - Anybody remember the pesky kanji from FM95? No? Well good, it was a pain and a half to figure out what those did... Differentials is the name that I give to the three options you have next to your buttons.  This is because they do just that, they change the input method of your command.  The first () refers to repeatedly pressing the key on the corresponding line, the second () refers to holding the key down, used for chargeup attacks like anything Guile can do.  The third one () is a bit of a mystery, however, if it's anything like FM95, it's a rotation differential.  It's used for 360° command input.  This is incredibly useful for command grapplers. Simply click on the blank box next to your buttons either once, twice, or three times to select a differential.


    Now we'll move on to the windows program icons.  Many of these are self explanatory, but for sake of completeness and lack of confusion, these icons are as follows:

    Animation Options - These are your basic function buttons used to adjust or test out animations.  BMP means Load Bitmap.  Pallet loads your characters palette bank so you can edit them,  Ani lets you view the animation of the currently selected skill, and Skill list loads up your characters entire skill list in a window to make navigation simpler.

    Red New Image Button - This button places an image before the box currently selected in the current skill using the image selected in the bitmap.

    Green New Image Button - This button places an image after the box currently selected in the current skill using the image selected in the bitmap.

    Blue Copy Button - This button replaces the currently selected image in the skill with the image selected in the bitmap.

    Image Select Buttons - These allow for various methods of selecting frames from a loaded bitmap file.   The first button selects the entire bitmap without shrinking the bounding box. The second button shrinks down the bounding box, essentially cropping the selected image so the edges of the sprites are the same as the bounding box.  This makes sure there's no wasted space in memory.  The third button is a bit of an oddity.  While it's clicked on (this is the default setting), the top left pixel of the selected area determines the transparency color.  If the button is clicked off, when you insert the bitmap into FM2nd, the sprite has no transparency... this is typically only used with backgrounds, as it produces ugly results with character sprites.

    Directory Upload Button - Quite a handy function for those of us who don't use sprite sheets.  What this button does is allow you to take an entire directory of 256 color bitmaps and load them into your character's image bank with one click.  It's a fantastic option if you don't feel like taking hours uploading bitmaps.  The only problem with this is you need to make sure each bitmap is perfect before you upload the directory, because otherwise you likely won't be able to find mistakes on your own once in the editor.

    Delete Unused Images Button - This is also a nice little feature.  Finish a character and aren't sure what frames of animation never got used??  No problem!  Click on this and any images loaded into the image bank that aren't actively being used by a skill will be deleted.  This will allow you to keep the filesize for your project down.

    Palette Banks - These buttons allow you to select which palette to edit.  They correspond to a specific player (1-8), more often than not, you won't see the extra palettes outside of Team Vs. Mode.
     
    Dark To Light - Clicking this button organizes your palette from darkest to lightest colors.  Typically though the results are frustratingly hard to work with, but if this is your cup of tea, by all means, go for it.

    Palette Order - Clicking this button arranges your color palettes from Monochrome, to Red, to Green, to Blue, in that order, with shades going from light to dark.  This is a fairly useful button for when your palette is a bit uncategorized.

    Edit - This button opens up the color editor.  I never find much need for it, as double clicking a color produces the same result.

    Animation Test Option Buttons - These are used in the Animation test to produce various ways of viewing the animations.  The first toggles whether or not to follow the character when movement is involved in the animation.  The second toggles collision detection boxes on or off.  The third toggles sounds on or off, and the fourth allow you to view an animation in half size in case it goes to high or moves too far out of the field of vision provided (or if you just have a REALLY big special effect).


    That's about it for the icons used in the program.  Hope this was informative!
